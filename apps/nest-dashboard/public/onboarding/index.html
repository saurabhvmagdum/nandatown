Four ways to contribute

Whoever you are, a contribution to NANDA Town can take any of these forms. You can pick one, or combine several. We prefer real code, but a working demo and a clear write-up count too.

1

Code contribution preferred

A GitHub pull request to NANDA Town, or a service plus a SkillMD that agents can call. This is the strongest kind of contribution.

2

A video of your demo

A short recording that shows your service working with NANDA Town. Great when your code is private but you still want to show what it does.

3

A live link to your demo

A public URL people can open and try themselves. Even better if an agent can reach it too, through an open API or hook.

4

A written case

A clear write-up of how your platform improves NANDA Town, and why a network of agents built on NANDA is a good ecosystem for what you do.

★Companies do not have to open-source their code. An open hook, an SDK, or API access is enough to plug into NANDA Town. Keep your IP, expose an interface agents can use.
